# 7.3 核心概念

> 理解 AgentX 的架构基石

本节将深入讲解 AgentX 的核心组件，这些组件共同构成了事件驱动智能体的运行基础。

---

## 7.3.1 AgentEngine：智能体引擎

### 什么是 AgentEngine

**AgentEngine** 是 AgentX 的核心引擎，负责处理智能体的所有状态转换和事件流。它实现了 Mealy Machine 模式，将 LLM 的异步响应转换为可预测的事件流。

```typescript
import { AgentEngine } from '@agentxjs/agent'

const engine = new AgentEngine({
  model: 'claude-sonnet-4-20250514',
  apiKey: process.env.ANTHROPIC_API_KEY
})
```

### 状态定义

AgentEngine 维护以下核心状态：

| 状态 | 说明 | 可转换到 |
|-----|------|---------|
| **idle** | 空闲，等待用户输入 | thinking |
| **thinking** | 思考中，处理 LLM 响应 | idle, tool_calling, error |
| **tool_calling** | 执行工具调用 | thinking, error |
| **error** | 错误状态 | idle |

### 状态转换图

```
┌──────────────────────────────────────────────────┐
│                                                   │
│    ┌───────┐   user_message   ┌──────────┐      │
│    │       │ ───────────────→ │          │      │
│    │ idle  │                  │ thinking │      │
│    │       │ ←─────────────── │          │      │
│    └───────┘   message_done   └────┬─────┘      │
│        ↑                           │            │
│        │       ┌───────────────────┘            │
│        │       │ tool_use                       │
│        │       ↓                                │
│        │   ┌──────────────┐                     │
│        │   │              │                     │
│        └───│ tool_calling │                     │
│ tool_result│              │                     │
│            └──────────────┘                     │
│                                                  │
└──────────────────────────────────────────────────┘
```

### 输入处理

AgentEngine 处理以下类型的输入：

```typescript
// 用户消息
engine.input({ type: 'user_message', content: '帮我写一段代码' })

// LLM 响应块
engine.input({ type: 'llm_chunk', delta: '好的，我来' })

// 工具调用请求
engine.input({ type: 'tool_use', name: 'search', input: { query: '...' } })

// 工具调用结果
engine.input({ type: 'tool_result', name: 'search', output: { ... } })

// LLM 响应结束
engine.input({ type: 'llm_end' })
```

### 输出事件

每次状态转换都会产生输出事件：

```typescript
engine.on('output', (events) => {
  for (const event of events) {
    switch (event.type) {
      case 'thinking_start':
        console.log('开始思考...')
        break
      case 'text_delta':
        process.stdout.write(event.delta)
        break
      case 'tool_start':
        console.log(`调用工具: ${event.name}`)
        break
      case 'tool_end':
        console.log(`工具返回: ${event.output}`)
        break
      case 'thinking_end':
        console.log('思考结束')
        break
      case 'message_complete':
        console.log(`消息完成: ${event.id}`)
        break
    }
  }
})
```

## 7.3.2 SystemBus：事件总线

### 什么是 SystemBus

**SystemBus** 是 AgentX 的中央事件总线，实现了发布-订阅模式。所有组件通过 SystemBus 进行通信，实现完全解耦。

```typescript
import { SystemBus } from '@agentxjs/runtime'

const bus = new SystemBus()
```

### 事件订阅

```typescript
// 订阅单个事件
bus.on('thinking_start', (event) => {
  console.log('AI 开始思考')
})

// 订阅多个事件
bus.on(['text_delta', 'tool_delta'], (event) => {
  console.log('收到增量数据')
})

// 订阅所有事件
bus.on('*', (event) => {
  console.log(`事件: ${event.type}`)
})

// 一次性订阅
bus.once('message_complete', (event) => {
  console.log('首条消息完成')
})
```

### 事件发布

```typescript
// 发布事件
bus.emit('user_message', {
  id: 'msg_001',
  content: '你好',
  timestamp: Date.now()
})

// 发布多个事件
bus.emitBatch([
  { type: 'thinking_start' },
  { type: 'text_delta', delta: '你好' },
  { type: 'thinking_end' }
])
```

### 事件过滤

```typescript
// 按类型过滤
const textEvents = bus.filter((event) => event.type === 'text_delta')

// 链式处理
bus.on('text_delta')
   .filter((e) => e.delta.length > 0)
   .map((e) => e.delta.toUpperCase())
   .subscribe((delta) => console.log(delta))
```

### 事件历史

```typescript
// 获取事件历史
const history = bus.getHistory()
console.log(`共 ${history.length} 个事件`)

// 重放事件
bus.replay(history)

// 清空历史
bus.clearHistory()
```

## 7.3.3 Driver 与 Presenter 模式

### 什么是 Driver/Presenter

AgentX 采用 **Driver/Presenter** 模式来分离输入处理和输出展示：

```
┌─────────────────────────────────────────────────────────┐
│                                                          │
│   ┌─────────┐      ┌───────────┐      ┌───────────┐   │
│   │         │ ───→ │           │ ───→ │           │   │
│   │ Driver  │      │ SystemBus │      │ Presenter │   │
│   │ (输入)  │      │  (事件)   │      │  (输出)   │   │
│   └─────────┘      └───────────┘      └───────────┘   │
│                                                          │
│   用户输入 ─────→ 事件流 ─────→ 界面展示                │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### Driver：输入驱动

Driver 负责将外部输入转换为系统事件：

```typescript
import { Driver } from '@agentxjs/runtime'

// CLI Driver
class CLIDriver extends Driver {
  constructor(bus: SystemBus) {
    super(bus)
  }

  start() {
    const rl = readline.createInterface({
      input: process.stdin,
      output: process.stdout
    })

    rl.on('line', (line) => {
      this.bus.emit('user_message', {
        id: generateId(),
        content: line,
        timestamp: Date.now()
      })
    })
  }
}

// HTTP Driver
class HTTPDriver extends Driver {
  start() {
    app.post('/chat', (req, res) => {
      this.bus.emit('user_message', {
        id: generateId(),
        content: req.body.message,
        timestamp: Date.now()
      })
    })
  }
}

// WebSocket Driver
class WebSocketDriver extends Driver {
  start() {
    wss.on('connection', (ws) => {
      ws.on('message', (data) => {
        const { content } = JSON.parse(data)
        this.bus.emit('user_message', {
          id: generateId(),
          content,
          timestamp: Date.now()
        })
      })
    })
  }
}
```

### Presenter：输出展示

Presenter 负责将事件转换为用户可见的输出：

```typescript
import { Presenter } from '@agentxjs/runtime'

// CLI Presenter
class CLIPresenter extends Presenter {
  constructor(bus: SystemBus) {
    super(bus)
    this.setup()
  }

  setup() {
    this.bus.on('thinking_start', () => {
      process.stdout.write('🤔 ')
    })

    this.bus.on('text_delta', (event) => {
      process.stdout.write(event.delta)
    })

    this.bus.on('thinking_end', () => {
      console.log('\n✅ 完成')
    })

    this.bus.on('error', (event) => {
      console.error(`❌ 错误: ${event.message}`)
    })
  }
}

// React Presenter
class ReactPresenter extends Presenter {
  private setMessages: React.Dispatch<...>

  setup() {
    this.bus.on('text_delta', (event) => {
      this.setMessages(prev => {
        const last = prev[prev.length - 1]
        return [...prev.slice(0, -1), {
          ...last,
          content: last.content + event.delta
        }]
      })
    })

    this.bus.on('message_complete', (event) => {
      this.setMessages(prev => [...prev, {
        id: event.id,
        role: 'assistant',
        content: event.content
      }])
    })
  }
}
```

### 优势

| 优势 | 说明 |
|-----|------|
| **解耦** | 输入和输出完全独立，可单独测试和替换 |
| **可组合** | 可以组合多个 Driver 和 Presenter |
| **可测试** | 通过 Mock 事件轻松测试各组件 |
| **可扩展** | 添加新的输入源或输出目标不影响其他代码 |

## 7.3.4 Agent 接口

### 创建 Agent

`createAgent` 是创建智能体的主要入口：

```typescript
import { createAgent } from '@agentxjs/runtime'

const agent = await createAgent({
  // 必需配置
  model: 'claude-sonnet-4-20250514',
  apiKey: process.env.ANTHROPIC_API_KEY,

  // 可选配置
  systemPrompt: '你是一个专业的助手',
  persistence: new SQLiteDriver('data.db'),
  mcpServers: [
    { name: 'filesystem', command: 'npx', args: [...] }
  ]
})
```

### Agent 方法

```typescript
// 同步聊天（等待完整响应）
const response = await agent.chat('你好')
console.log(response)

// 流式聊天
for await (const chunk of agent.stream('给我讲个故事')) {
  process.stdout.write(chunk)
}

// 事件监听
agent.on('thinking_start', () => { ... })
agent.on('text_delta', (e) => { ... })
agent.on('tool_start', (e) => { ... })
agent.on('message_complete', (e) => { ... })

// 获取历史
const messages = await agent.getMessages()

// 清空历史
await agent.clearMessages()

// 销毁智能体
await agent.destroy()
```

### 高级配置

```typescript
const agent = await createAgent({
  model: 'claude-sonnet-4-20250514',
  apiKey: process.env.ANTHROPIC_API_KEY,

  // 模型参数
  maxTokens: 4096,
  temperature: 0.7,

  // 重试配置
  retry: {
    maxRetries: 3,
    retryDelay: 1000
  },

  // 超时配置
  timeout: 60000,

  // 钩子函数
  hooks: {
    beforeRequest: async (messages) => {
      console.log('请求前:', messages.length, '条消息')
      return messages
    },
    afterResponse: async (response) => {
      console.log('响应后:', response.usage)
      return response
    }
  }
})
```

## 7.3.5 组件协作

### 完整的事件流

```
用户输入 "写一段代码"
    │
    ↓
┌──────────────────────────────────────────────────────────┐
│  Driver                                                   │
│  将输入转换为 user_message 事件                           │
└──────────────────────────────────────────────────────────┘
    │
    ↓ emit('user_message', {...})
┌──────────────────────────────────────────────────────────┐
│  SystemBus                                                │
│  广播事件到所有订阅者                                      │
└──────────────────────────────────────────────────────────┘
    │
    ↓ on('user_message')
┌──────────────────────────────────────────────────────────┐
│  AgentEngine                                              │
│  状态转换: idle → thinking                                │
│  输出事件: thinking_start                                 │
│  调用 LLM API                                             │
└──────────────────────────────────────────────────────────┘
    │
    ↓ emit('thinking_start'), emit('text_delta'), ...
┌──────────────────────────────────────────────────────────┐
│  SystemBus                                                │
│  广播事件                                                  │
└──────────────────────────────────────────────────────────┘
    │
    ↓ on('text_delta')
┌──────────────────────────────────────────────────────────┐
│  Presenter                                                │
│  显示流式文本                                              │
└──────────────────────────────────────────────────────────┘
    │
    ↓
用户看到 AI 逐字输出响应
```

### 代码示例

```typescript
import { createAgent, SystemBus, CLIDriver, CLIPresenter } from '@agentxjs/runtime'

async function main() {
  // 创建事件总线
  const bus = new SystemBus()

  // 创建智能体
  const agent = await createAgent({
    model: 'claude-sonnet-4-20250514',
    apiKey: process.env.ANTHROPIC_API_KEY,
    bus // 共享事件总线
  })

  // 创建输入驱动
  const driver = new CLIDriver(bus)

  // 创建输出展示
  const presenter = new CLIPresenter(bus)

  // 启动
  console.log('智能体已启动，请输入消息：')
  driver.start()
}

main()
```

## 本节要点

1. **AgentEngine**：基于 Mealy Machine 的状态机引擎
2. **SystemBus**：发布-订阅模式的事件总线
3. **Driver/Presenter**：分离输入处理和输出展示
4. **Agent 接口**：统一的智能体创建和操作 API
5. **组件协作**：通过事件流实现松耦合的组件通信
