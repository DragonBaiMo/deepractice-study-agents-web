> 回顾与展望

## 7.6.1 核心概念回顾

### 事件驱动架构

AgentX 的核心设计理念是**事件驱动**：

```
传统模式：request → wait → response（同步阻塞）
AgentX：  event → process → event（异步非阻塞）
```

**优势**：
- 松耦合：组件通过事件通信，互不依赖
- 可观测：所有事件可追踪、可重放
- 可扩展：添加新功能不影响现有代码
- 实时性：流式输出，即时反馈

### Mealy Machine 模式

AgentEngine 采用 Mealy Machine 状态机：

```
(state, input) → (state, outputs)

示例：
(idle, user_message) → (thinking, [thinking_start])
(thinking, llm_chunk) → (thinking, [text_delta])
(thinking, llm_end) → (idle, [thinking_end, message_complete])
```

**特点**：
- 纯函数：相同输入总是相同输出
- 可测试：状态转换可单元测试
- 可预测：行为完全由状态和输入决定

### 四层事件模型

| 层级 | 事件类型 | 频率 | 用途 |
|-----|---------|-----|------|
| **Stream** | text_delta, tool_delta | 高 | 实时流式输出 |
| **State** | thinking_start/end, tool_start/end | 中 | 状态变化通知 |
| **Message** | user_message, assistant_message | 低 | 完整消息记录 |
| **Turn** | turn_complete | 最低 | 轮次统计分析 |

## 7.6.2 架构组件总结

### 核心组件

| 组件 | 职责 | 关键方法 |
|-----|------|---------|
| **AgentEngine** | 状态机引擎 | input(), on('output') |
| **SystemBus** | 事件总线 | emit(), on(), filter() |
| **Container** | 生命周期管理 | register(), create(), destroy() |
| **Environment** | 外部系统接口 | llm.chat(), mcp.callTool() |
| **Persistence** | 数据持久化 | saveMessage(), getMessages() |

### Driver/Presenter 模式

```
Driver（输入）→ SystemBus（事件）→ Presenter（输出）
```

- **Driver**：将外部输入转换为事件（CLI、HTTP、WebSocket）
- **Presenter**：将事件转换为可见输出（终端、React UI）

## 7.6.3 与 PromptX 集成

### 集成方式

通过 MCP 协议将 PromptX 作为工具服务器接入：

```typescript
mcpServers: [
  { name: 'promptx', command: 'npx', args: ['-y', '@promptx/cli'] }
]
```

### 认知能力

| 能力 | PromptX 工具 | 作用 |
|-----|-------------|------|
| **角色激活** | action | 激活专业角色，获得领域知识 |
| **工具调用** | toolx | 使用 Luban 创建的工具 |
| **记忆检索** | recall | 从记忆网络获取相关信息 |
| **记忆存储** | remember | 保存新知识到记忆网络 |
| **资源发现** | discover | 发现可用的角色和工具 |

### 认知循环

```
recall → 思考 → remember
  ↑                 │
  └─────────────────┘
```

## 7.6.6 快速参考

### 快速启动

```bash
# 方式一：NPX
export ANTHROPIC_API_KEY=sk-ant-xxx
npx @agentxjs/portagent

# 方式二：Docker
docker run -d -e ANTHROPIC_API_KEY=sk-ant-xxx -p 5200:5200 agentxjs/portagent
```

### 代码模板

```typescript
import { createAgent } from '@agentxjs/runtime'

const agent = await createAgent({
  model: 'claude-sonnet-4-20250514',
  apiKey: process.env.ANTHROPIC_API_KEY,
  mcpServers: [
    { name: 'promptx', command: 'npx', args: ['-y', '@promptx/cli'] }
  ]
})

agent.on('text_delta', (e) => process.stdout.write(e.delta))
agent.on('thinking_start', () => console.log('🤔'))
agent.on('thinking_end', () => console.log('✅'))

await agent.chat('你好')
```

### 关键 API

```typescript
// 创建智能体
createAgent(config): Promise<Agent>

// Agent 方法
agent.chat(message): Promise<string>
agent.stream(message): AsyncGenerator<string>
agent.on(event, handler): void
agent.getMessages(): Promise<Message[]>
agent.destroy(): Promise<void>

// SystemBus 方法
bus.emit(type, data): void
bus.on(type, handler): void
bus.filter(predicate): Observable
```
