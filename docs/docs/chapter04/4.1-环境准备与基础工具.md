# 4.1 环境准备与基础工具

在开始构建智能体之前，我们需要搭建开发环境并定义一些基础组件。这些准备工作将在后续实现不同范式时被复用。

---

## 4.1.1 安装依赖

本章使用 Python 语言，建议使用 Python 3.10 或更高版本。

```bash
pip install openai python-dotenv
```

- `openai`：用于调用大语言模型 API
- `python-dotenv`：用于管理环境变量

---

## 4.1.2 配置 API 密钥

在项目根目录创建 `.env` 文件，配置模型服务信息：

```bash
# .env file
LLM_API_KEY="YOUR-API-KEY"
LLM_MODEL_ID="YOUR-MODEL"
LLM_BASE_URL="YOUR-URL"
```

这种配置方式的好处是：
1. 敏感信息不会硬编码在代码中
2. 可以方便地切换不同的模型服务
3. 支持 OpenAI 官方服务或任何兼容接口的第三方服务

> 如果不确定如何获取 API 密钥，可参考 [Datawhale API设置教程](https://datawhalechina.github.io/handy-multi-agent/#/chapter1/1.2.api-setup)

## 4.1.3 封装 LLM 客户端

为了让代码更清晰、更易复用，我们封装一个专用的 LLM 客户端类：

```python
import os
from openai import OpenAI
from dotenv import load_dotenv
from typing import List, Dict

# 加载 .env 文件中的环境变量
load_dotenv()

class HelloAgentsLLM:
    """
    为本书定制的LLM客户端。
    支持任何兼容OpenAI接口的服务，默认使用流式响应。
    """
    def __init__(self, model: str = None, apiKey: str = None, baseUrl: str = None, timeout: int = None):
        """
        初始化客户端。优先使用传入参数，否则从环境变量加载。
        """
        self.model = model or os.getenv("LLM_MODEL_ID")
        apiKey = apiKey or os.getenv("LLM_API_KEY")
        baseUrl = baseUrl or os.getenv("LLM_BASE_URL")
        timeout = timeout or int(os.getenv("LLM_TIMEOUT", 60))

        if not all([self.model, apiKey, baseUrl]):
            raise ValueError("模型ID、API密钥和服务地址必须提供或在.env文件中定义。")

        self.client = OpenAI(api_key=apiKey, base_url=baseUrl, timeout=timeout)

    def think(self, messages: List[Dict[str, str]], temperature: float = 0) -> str:
        """
        调用大语言模型进行思考，返回响应文本。

        参数:
        - messages: 消息列表，格式为 [{"role": "user", "content": "..."}]
        - temperature: 温度参数，0表示确定性输出

        返回:
        - 模型响应的完整文本
        """
        print(f"🧠 正在调用 {self.model} 模型...")
        try:
            response = self.client.chat.completions.create(
                model=self.model,
                messages=messages,
                temperature=temperature,
                stream=True,
            )

            # 处理流式响应
            print("✅ 大语言模型响应成功:")
            collected_content = []
            for chunk in response:
                content = chunk.choices[0].delta.content or ""
                print(content, end="", flush=True)
                collected_content.append(content)
            print()  # 换行
            return "".join(collected_content)

        except Exception as e:
            print(f"❌ 调用LLM API时发生错误: {e}")
            return None
```

**使用示例：**

```python
if __name__ == '__main__':
    llmClient = HelloAgentsLLM()

    messages = [
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "写一个快速排序算法"}
    ]

    response = llmClient.think(messages)
```

## 4.1.4 定义工具执行器

智能体需要调用外部工具来完成任务。我们创建一个通用的工具管理器：

```python
from typing import Dict, Any

class ToolExecutor:
    """
    工具执行器，负责管理和执行工具。
    """
    def __init__(self):
        self.tools: Dict[str, Dict[str, Any]] = {}

    def registerTool(self, name: str, description: str, func: callable):
        """
        注册一个新工具。

        参数:
        - name: 工具名称，供智能体调用
        - description: 工具描述，帮助智能体判断何时使用
        - func: 工具的执行函数
        """
        if name in self.tools:
            print(f"警告: 工具 '{name}' 已存在，将被覆盖。")
        self.tools[name] = {"description": description, "func": func}
        print(f"工具 '{name}' 已注册。")

    def getTool(self, name: str) -> callable:
        """根据名称获取工具的执行函数。"""
        return self.tools.get(name, {}).get("func")

    def getAvailableTools(self) -> str:
        """获取所有可用工具的格式化描述。"""
        return "\n".join([
            f"- {name}: {info['description']}"
            for name, info in self.tools.items()
        ])
```

一个良好定义的工具需要包含三个要素：

| 要素 | 说明 | 示例 |
|------|------|------|
| **名称** | 简洁的唯一标识符 | `Search` |
| **描述** | 说明工具用途，供模型判断 | "搜索网络信息" |
| **执行函数** | 实际执行任务的代码 | `search(query)` |

其中，**描述是最关键的部分**——大语言模型依赖这段描述来判断何时使用哪个工具。

## 4.1.5 实现搜索工具

作为示例，我们实现一个基于 SerpApi 的网页搜索工具：

```bash
pip install google-search-results
```

在 `.env` 中添加 API 密钥：

```bash
SERPAPI_API_KEY="YOUR_SERPAPI_API_KEY"
```

工具实现：

```python
from serpapi import SerpApiClient

def search(query: str) -> str:
    """
    网页搜索工具。
    智能解析搜索结果，优先返回直接答案或知识图谱信息。
    """
    print(f"🔍 正在搜索: {query}")
    try:
        api_key = os.getenv("SERPAPI_API_KEY")
        if not api_key:
            return "错误: SERPAPI_API_KEY 未配置。"

        params = {
            "engine": "google",
            "q": query,
            "api_key": api_key,
            "gl": "cn",
            "hl": "zh-cn",
        }

        client = SerpApiClient(params)
        results = client.get_dict()

        # 智能解析：优先返回直接答案
        if "answer_box" in results and "answer" in results["answer_box"]:
            return results["answer_box"]["answer"]
        if "knowledge_graph" in results and "description" in results["knowledge_graph"]:
            return results["knowledge_graph"]["description"]
        if "organic_results" in results and results["organic_results"]:
            # 返回前三个搜索结果的摘要
            snippets = [
                f"[{i+1}] {res.get('title', '')}\n{res.get('snippet', '')}"
                for i, res in enumerate(results["organic_results"][:3])
            ]
            return "\n\n".join(snippets)

        return f"未找到关于 '{query}' 的信息。"

    except Exception as e:
        return f"搜索错误: {e}"
```

## 4.1.6 测试基础组件

将搜索工具注册到执行器，并进行测试：

```python
if __name__ == '__main__':
    # 1. 初始化工具执行器
    toolExecutor = ToolExecutor()

    # 2. 注册搜索工具
    toolExecutor.registerTool(
        "Search",
        "网页搜索引擎。用于查询时事、事实等知识库之外的信息。",
        search
    )

    # 3. 查看可用工具
    print("\n--- 可用工具 ---")
    print(toolExecutor.getAvailableTools())

    # 4. 测试工具调用
    print("\n--- 测试搜索 ---")
    tool = toolExecutor.getTool("Search")
    result = tool("英伟达最新的GPU型号")
    print(result)
```
